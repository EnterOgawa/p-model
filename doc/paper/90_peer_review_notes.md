目的：
- Phase 8（論文化）で想定される批判点を先に列挙し、本文の表現・注記・限界の明文化を「後追い」ではなく前倒しで行う。
- 「本文に反映すべき応答」と「メモに留める応答」を分離し、更新漏れを防ぐ。

注記：

## A. 想定される批判点（リスト）

### C1：P(x) の動力学（場の方程式）が無い／恣意的に見える
- 指摘例：P の支配方程式、エネルギー保存、物質との結合が未提示。結果の再現はできても、理論として閉じていない。

### C2：GR との関係が曖昧（「置換」なのか「有効理論」なのか）
- 指摘例：GR を近似と呼ぶなら、どの極限で一致し、どの極限で差が出るかを定量的に示すべき。

### C3：自由度（β, δ）の導入が ad hoc に見える
- 指摘例：β や δ は、観測に合わせたパラメータに見える。一次ソースと独立に固定できるのか。

### C4：同一パラメータで複数テストを貫けているか（テストごとの調整疑い）
- 指摘例：データセットごとに条件を変えれば一致するのは当然。凍結パラメータと“動かしたもの”を明示すべき。

### C5：弱場テストでの独立性（GR の前処理・補正に依存していないか）
- 指摘例：Cassini/GPS/LLR の処理が暗黙に GR 前提（時計系・座標系・補正）へ依存していないか。

### C6：強場（EHT/S2）での系統誤差が支配的で、差分予測が読めない
- 指摘例：リング直径・散乱・放射モデル・スピン/傾斜の系統が大きく、係数差が識別不能では。

### C7：重力波（chirp）の扱いが簡易すぎる
- 指摘例：公開 strain からの単純抽出は、公式テンプレート解析の代替ではない。結論の位置づけを厳密にすべき。

### C8：宇宙論（距離二重性）の張力は “棄却” ではないのか
- 指摘例：SNIa+BAO の DDR 制約で強く棄却されるなら、モデルは否定的に見える。何を主張しているのか曖昧。

### C9：Step 4.7（再接続条件）の物理的意味が読み取れない
- 指摘例：α, s_L, s_R は何を表すのか。どの仮定を緩めれば整合するのかが本文から分かりにくい。

### C10：BAO（一次統計）での推定量・共分散・窓関数の扱いが複雑
- 指摘例：推定量の定義差、recon の仕様差、window mixing、cov 推定法の差で結論が揺れていないか。

### C11：再現性はあるが「反証可能性（棄却条件）」が読者に伝わりにくい
- 指摘例：何が観測されれば棄却か、どの測定精度で決着するか、が本文の主張として弱い。

### C12：スコープが広すぎる（量子まで含めると散漫）
- 指摘例：量子現象まで言及すると、宇宙物理の検証が弱く見える。到達点と未解決点の線引きが必要。

### C13：図の可読性（凡例かぶり、はみ出し、図番号）が査読で落とされる
- 指摘例：図が読めないと評価不能。図番号の参照が不安定だと混乱する。

### C14：一次ソースの固定が不十分に見える
- 指摘例：参照日・取得元・ローカルキャッシュ・ハッシュなど、再現に必要な情報の抜けがないか。

## B. 応答（本文/メモへの反映計画）

※この節は各 C# に対して「本文へ反映する文言」「メモで補足する内容」「対応先（節/出力）」を整理する。

### R1（C1）：P(x) の動力学（場の方程式）について
- 本文：本稿は「P→観測量の写像（重力/時計/光）」の検証を目的とする **有効モデル**であり、P の支配方程式（場の方程式）やエネルギー保存の導出は未解決であることを明記する（結果の位置づけを誤解させない）。
- メモ：場の方程式は、(i) 一般共変な作用から導く案、(ii) エネルギー/運動量の局所保存を満たす最小方程式、(iii) 量子（位相・干渉）と整合する波動方程式、のいずれかで閉じる必要がある（Phase 7/将来課題）。
- 対応先：`doc/paper/10_manuscript.md`（新設 1.3 節）

### R2（C2）：GR との関係（置換か有効理論か）
- 本文：GR/SR と P-model は「同じ観測結果へ至る記述（近似）と機構（時間波）の違い」であることを再掲し、**弱場で一致・強場/宇宙論で差が出る**という立場を「主張の範囲」として明文化する。
- メモ：一致の理由（ニュートン極限・PPN対応・光学極限）と、差分予測の入口（EHT 係数差、速度飽和 `δ_0`（拡張仮説）、距離指標前提の再接続条件）を“決着可能な観測”として列挙する。
- 対応先：`doc/paper/10_manuscript.md`（1.1/新設 1.3 節）

### R3（C3）：自由度（β, δ）の ad hoc 性
- 本文：β は Cassini の PPN γ 制約を一次ソースとして凍結し、テスト毎の調整をしない。速度飽和 `δ_0` は Part I のコア写像には含めず、拡張仮説として上限拘束と棄却条件の形を固定する（“後からフィット”はしない）。
- メモ：凍結値と一次ソースを機械可読に固定している（`output/theory/frozen_parameters.json` は β、`output/theory/delta_saturation_constraints.json` は `δ_0` の上限例）。将来、理論側（場方程式）から β/`δ_0` の値域が導けるかが次段。
- 対応先：`doc/paper/10_manuscript.md`（新設 1.3 節、2.4 節の注記）／`output/theory/frozen_parameters.json`／`output/theory/delta_saturation_constraints.json`

### R4（C4）：同一パラメータで複数テストを貫けているか
- 本文：本稿の方針（fit と predict の分離）を本文に明文化し、どの結果が「凍結パラメータの外挿（predict）」で、どこが「前段の探索/スクリーニング（fit）」かを読者が判別できるようにする。
- メモ：弱場の統合出力として “何を固定し、何を動かしたか” を機械可読で固定している（`output/summary/weak_field_test_matrix.json` など）。
- 対応先：`doc/paper/10_manuscript.md`（新設 1.3 節、4.1.1 付近の注記）／`output/summary/weak_field_test_matrix.json`

### R5（C5）：弱場テストが GR の前処理に依存していないか
- 本文：一次データでも「公的プロダクト」が GR/SR の時刻系や補正モデルを含む場合があることを明記し、本稿の主張が「完全な生データ再解析」ではなく「公開一次ソースでの再現可能な差分検証」であることを明確化する。
- メモ：生データ（受信ログ/観測方程式）レベルの完全再解析は今後の課題として切り分け、現状は (i) 入力の一次ソースを固定、(ii) 補正の内訳を出力化、(iii) 系統テンプレを用意、で“隠れ前提”を可視化する方針とする。
- 対応先：`doc/paper/10_manuscript.md`（新設 1.3 節、付録/再現性の注記）

### R6（C6）：強場（EHT/S2）での系統誤差が支配的
- 本文：強場は「差分予測そのもの」だけでなく「差分が判別可能になるために必要な精度（誤差予算）」が重要であり、現状は M87* は質量/距離、Sgr A* は ring σ と κ 系統が律速である、という位置づけを明文化する。
- メモ：EHT は “決定的検証候補” として、必要精度を固定出力し、改善が達した時点で判定できる形にする（`output/summary/decisive_candidates.json` / `output/summary/decisive_falsification.json`）。S2 は一次ソース固定後に強場差分予測（係数差）へ接続する（Phase 5/8 の追加課題）。
- 対応先：`doc/paper/10_manuscript.md`（EHT 節の注記）／`output/summary/decisive_candidates.json`

### R7（C7）：重力波（chirp）の扱いが簡易すぎる
- 本文：GW150914 の扱いは、公式のテンプレート解析を置換するものではなく、公開 strain からの **整合性チェック（sanity check）**として位置づけることを明記する（結論を過大に見せない）。
- メモ：将来、公開 posterior（LIGO/Virgo/KAGRA）や再解析フレームワークに接続し、P-model 予測との差分（位相、偏極、放射反作用）を定量化する必要がある（Phase 3/7 の延長）。
- 対応先：`doc/paper/10_manuscript.md`（GW 節の冒頭注記）

### R8（C8）：宇宙論（DDR）張力は “棄却” ではないのか
- 本文：「静的背景Pの最小仮定」は DDR/BAO で強い張力を持つことを明記しつつ、解釈を (A) 仮定の棄却 と (B) 距離指標前提の再検証 に分け、後者を **必要補正（再接続条件）**として定量化している点を明確化する。
- メモ：本稿は“救済”ではなく「必要条件を固定し、独立制約（不透明度・SNe進化・BAO自己整合）で反証できる形にする」ことを目的とする。張力が残る場合は、そのまま棄却（A）として扱える。
- 対応先：`doc/paper/10_manuscript.md`（2.5.3 節の冒頭/新設要約）

### R9（C9）：Step 4.7（再接続条件）の物理的意味が読めない
- 本文：α（不透明度）、s_L（標準光源の光度進化）、s_R（標準定規のスケール進化）、p_t（時間伸長）、p_e（光子エネルギー）を **“どの観測量をどう変えるか”**として定義し、判定基準（|z|<3/5）と、d_L/d_A 寄与分解（z=1 anchor）を本文に追記する。
- メモ：再接続条件は “機構の選択” ではなく “必要な組合せ” を固定する出力であり、各機構は独立制約で z スコア評価する（出力：`output/cosmology/cosmology_ddr_reconnection_conditions_metrics.json`）。
- 対応先：`doc/paper/10_manuscript.md`（2.5.3 節）／`output/cosmology/cosmology_ddr_reconnection_conditions_metrics.json`

### R10（C10）：BAO の推定量・共分散・窓関数が複雑
- 本文：BAO は推定量/窓関数/再構成/共分散で系統が入り得るため、本稿では「一次統計（銀河+random）→ ξℓ → peakfit」の再現性確保を優先し、公開圧縮出力（D_M,H）に依存する議論は “前提の検証（Step 4.7）” として扱うことを明確化する。
- メモ：DESI DR1 の raw から ξ0/ξ2+cov を自前生成し ε fit まで到達することが、公開圧縮出力依存の議論を外す鍵となる（Phase 4.5 の未完了点）。
- 対応先：`doc/paper/10_manuscript.md`（2.5.1/2.5.3 の注意）／`doc/ROADMAP.md`（Step 4.5）

### R11（C11）：反証可能性（棄却条件）が伝わりにくい
- 本文：反証条件を「結果の一覧」ではなく本文の主張として前倒しで提示し、どの精度で決着するか（EHT など）も併記する。
- メモ：反証条件は機械可読で固定している（`output/summary/decisive_falsification.json`、弱場：`output/summary/weak_field_falsification.json`）。
- 対応先：`doc/paper/10_manuscript.md`（新設 1.3 節、4.1.1 の注記）

### R12（C12）：スコープが広すぎる（量子まで含めると散漫）

- メモ：量子は Phase 7 以降で一次データと結びつける（ベルテスト等）。現段階では“動機付け”以上の主張にしない。
- 対応先：`doc/paper/10_manuscript.md`（1.1/新設 1.3 節）
